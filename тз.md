# **Техническое задание для AI-агента (Разработка веб\-сайта)**

## **1\. Концепция и визуальный характер**

* **Эстетика:** Экстремальный текстовый минимализм (90% контента — типографика).  
* **Стилистика:** Имитация чистой, строгой статьи в духе **Notion** или профессионального лонгрида. Полный отказ от крупных, кричащих заголовочных шрифтов и декоративных интерфейсных элементов.  
* **Движок:** Next.js (Vercel), React, Tailwind CSS.  
* **Анимация:** Плавный скролл (Smooth scrolling) и фоновый WebGL-контейнер (Three.js) на заднем плане под контентом.

## **2\. Дизайн-система (Строгие ограничения)**

### **Типографика**

* **Шрифт:** Inter для всего сайта.  
* **Параметры:** Отрицательный трекинг (letter-spacing: \-0.02em или tracking-tight), плотный межстрочный интервал (line-height).  
* **Иерархия размеров (Максимальная скромность):**  
  * **Имя (Главный заголовок):** Не более 24px / text-2xl, полужирный (font-semibold).  
  * **Названия блоков (Услуги, Кейсы):** Не выделять крупным размером. Использовать размер основного текста 16px (text-base), но с насыщенностью font-semibold и, возможно, верхним регистром (Uppercase).  
  * **Основной текст и списки:** Стандартный журнальный размер 15px–16px (text-base), регулярное начертание.

### **Цветовая палитра и темы (Light / Dark)**

Переключатель тем (Toggle) должен быть реализован в виде аккуратного текстового линка в углу экрана.

| Элемент | Дневная тема (По умолчанию) | Ночная тема |
| :---- | :---- | :---- |
| **Общий фон сайта** | \#E8EFF2 | \#0D1113 |
| **Центральный шейп (Контент)** | \#E8EFF2 | \#161B1E |
| **Основной текст** | \#000000 | \#FFFFFF |
| **Акцент (Ссылки/Интерактив)** | \#FD4B32 | \#FD4B32 |

## **3\. Сетка и Layout (Структура Notion-статьи)**

* **Центрирование:** Весь контент выстроен в одну строгую вертикальную колонку по центру экрана (максимальная ширина текстового контейнера — 680px–720px, как в Notion).  
* **Эффект шейпа:** Контентная колонка визуально лежит в едином пространстве (шейпе) \#E8EFF2.  
* **Разделители:** Между логическими блоками вместо массивных отступов использовать тонкие аккуратные линии в 1 пиксель или едва заметные отступы, характерные для абзацев текстового редактора.  
* **Задний план:** Полноэкранный canvas (z-index: 1\) с прозрачным фоном для будущей 3D-анимации. Контентный слой находится поверх него (z-index: 2).

## **5\. Технические требования к агенту**

1. **Архитектура тем:** Реализовать переключение через корневой класс темы без скачков цвета при перезагрузке страницы.  
2. **Заглушка Three.js:** Инициализировать базовую сцену с прозрачным бэкграундом и элементарным мешем (например, медленно вращающимся тонким каркасным объектом), чтобы подготовить рабочее окружение для финальной WebGL графики.  
3. **Адаптивность:** Обеспечить строгий перенос строк и сохранение компактных отступов на мобильных экранах. Контент не должен расползаться по ширине, удерживая форму аккуратной текстовой колонки.
